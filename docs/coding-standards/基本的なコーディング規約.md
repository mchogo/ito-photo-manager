# 基本的なコーディング規約

## 共通事項

- **コメント**: 変数名や関数名から意図が読み取れない場合のみコメントを付ける。不要な `TODO` や `FIXME` を放置しない。
- **ファイルサイズ**: 単一ファイルが 400行を超えたら責務分割を検討する。
- **テストの信頼性**: `time.sleep` や固定タイムスタンプに依存するテストは避け、モックやフィクスチャを活用する。

## Python (Backend)

| 項目 | 規約 |
|---|---|
| 変数・関数 | `snake_case` |
| クラス | `PascalCase` |
| 定数 | `UPPER_SNAKE_CASE` |
| Linter | `ruff` (`.ruff.toml` 参照) |
| Formatter | `ruff format` を推奨 |
| 型ヒント | 全関数に付与。`Any` は原則禁止 |
| print | 禁止。`logging` モジュールを使用 |

## JavaScript / TypeScript (Frontend)

| 項目 | 規約 |
|---|---|
| 変数・関数 | `camelCase` |
| React コンポーネント / Type / Interface | `PascalCase` |
| ページファイル | `kebab-case` |
| エクスポート定数 | `UPPER_SNAKE_CASE` |
| Linter | `next lint` (ESLint) |
| Formatter | Prettier を推奨 |

## Next.js App Router 固有ルール

- `page.tsx` はルーティング・レイアウトに留め、ビジネスロジックはフックやモジュールに分離する。
- API 通信は `lib/api.ts` の共通ラッパーを経由し、コンポーネント内での直接 `fetch` を避ける。
