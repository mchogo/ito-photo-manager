# 改善要件定義：いとうさんフォトマネージャー UI/UXブラッシュアップ

## 1. 現状の課題と改善方針
- **UX: ナビゲーションの直感性**: どの画面からでもトップに戻れるよう、ヘッダータイトルをリンク化する。
- **UI: モバイル表示の不具合**: 撮影画面において、特定条件下でカメラ起動ボタンやUIの一部が隠れる（iOSのセーフエリアや固定バーの干渉）問題を解消する。
- **UX: 現場での使い勝手**: 通信環境や操作ミスを考慮したフィードバックの強化。

## 2. 具体的な修正項目
### A. ヘッダーの改善
- `frontend/app/layout.tsx` のタイトル部分を `next/link` で `/` に紐付ける。
- クリック可能であることがわかるよう、ホバー時やタップ時にわずかに透明度を変える等のフィードバックを追加。

### B. 撮影画面（shoot/page.tsx）の修正
- **下部ボタンの重なり解消（重要）**: 
  - スクリーンショットで確認された通り、最下部の固定バーが最後の機器スロットの操作を妨げています。`pb-[200px]` 程度の十分な余白を確保し、すべてのUIが干渉なく操作できる状態にします。
- **離脱時のセッション警告**:
  - ヘッダーリンクやブラウザバックでページを離れる際、未提出のデータがある場合は確認ダイアログを表示し、不意なデータ消失を防ぎます。

### C. 総合的なUX向上・データ永続化
- **入力情報のオート保存（オートコンプリート）**: 
  - トップ画面（`page.tsx`）の入力項目（現場ID等）を `localStorage` に保持し、作業の中断・再開時に前回のデータが自動入力されるようにします。
- **インタラクションの洗練**: 
  - 写真アップロード時のスケルトン表示や、ボタンのタップアニメーションを統一し、プレミアムな操作感を実現します。

### プロンプト：UI/UX設計の最終ブラッシュアップと不具合修正

「現在実装されている『いとうさんフォトマネージャー』に対して、以下の3つの重大な修正と機能追加を行ってください。

**1. 下部ボタンの重なり解消（クリティカル）**
- スクリーンショットにある通り、画面下部の『残りX枚』ボタン（fixed）が、最後にある撮影スロットのボタンを覆い隠して操作できなくなっています。
- スクロール領域の最下部に十分なパディング（`padding-bottom: 200px` 程度）を追加し、すべてのボタンが確実に独立して見えるようにレイアウトを修正してください。

**2. 意図しない離脱への警告**
- ヘッダーのロゴクリックやブラウザの戻る操作でトップページに戻る際、現在進行中の撮影データがある場合は『作業を中断してトップに戻りますか？現在のセッション内容は失われます（提出した写真は保存されています）』という警告ダイアログを出してください。

**3. 入力内容の永続化（オートコンプリート）**
- トップページ（`page.tsx`）の『現場ID』や『作業員名』などの入力項目は、入力のたびに `localStorage` に保存してください。
- 次回アクセス時や、撮影画面から戻ってきた際に、前回の入力内容が自動で補完されている状態にしてください。

これらの修正を反映した、`page.tsx`, `layout.tsx`, `shoot/page.tsx` の修正後コードを提示してください。」
