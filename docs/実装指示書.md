# 要件定義および実装指示書：現場撮影管理・Excel自動化システム「いとうさんフォトマネージャー」

## 1. サマリ
- **目的**: 現場でのPOSレジ・通信機器の撮影漏れを防止し、提出用Excelの作成を自動化する。
- **スコープ**: 
  - 現場作業員向けのWebアプリ/ツール（機器チェックから撮影まで）
  - 管理者向けのExcel報告書自動生成機能
- **成果物**: 
  - 撮影管理Webアプリ（要件・仕様）
  - Excel出力スクリプト（要件・仕様）
- **想定ユーザ**: 現場施工スタッフ、事務管理担当

## 2. 要件（機能要件）
### ユースケース
- **[作業開始]**: 現場に到着し、導入する機器のリストをチェックボックスで選択する。
- **[撮影枠生成]**: 選択された機器に基づき、アプリが「撮影すべき項目」の枠（スロット）を動的に生成する。
- **[撮影・登録]**: 各枠に対してカメラを起動し、写真を撮影、またはフォルダから選択して登録する。
- **[完了確認]**: 全ての枠が埋まっていない場合、警告を表示し提出を防ぐ。
- **[データ出力]**: 登録された写真と情報を管理側へ送り、Excelの特定セル（あるいはテンプレート）へ自動配置する。

### 画面/操作案（UIデザイン：iOS Liquid Glass風）
- **デザインコンセプト**: iOSのコントロールセンターのような「グラスモフィズム（透過・ぼかし・光沢）」を採用。
- **背景**: ぼかしの効いたグラデーション背景。
- **カード**: 背面が透けて見える半透明の白（ガラス）、細い白のボーダー、ドロップシャドウ。
- **ボタン**: 液体のような弾力のあるアニメーション（Active時）、鮮やかなアクセントカラー。

## 3. データ要件
### データモデル案
- **マスター**: 機器種別、標準撮影項目（POSなら3点、ルーターなら2点など）
- **案件データ**: 現場ID、作業日、作業員名
- **写真データ**: 写真バイナリ、紐付け機器ID、撮影日時

## 4. 外部IF・連携
- **Excel連携**: Pythonの `openpyxl` または `pandas`、あるいは Google Apps Script (GAS) を使用。
- **保存先**: クラウドストレージ（Google Drive, Dropbox等）への自動保存。

## 5. 制約・前提・リスク
- **簡易性**: 現場作業員は手袋をしていたり忙しいため、極力タップ数を減らす。
- **オフライン対応**: 地下や建物の奥など電波が悪い現場も想定し、一時的にローカル保存し、電波復帰時に一括送信する仕組みが望ましい。

## 6. Claude向け実装指示書

### 推奨技術スタック
- **フロントエンド**: Streamlit (Pythonのみで完結、簡易UIに最適) または Next.js (リッチなUIが必要な場合)
- **バックエンド/ストレージ**: Supabase (DB + Storage) または Google Apps Script + Google Drive
- **Excel出力**: Python (`openpyxl`) または GAS

### 改修・実装タスク
1.  **機器マスターの定義**: 主要機器（POS、ルーター等）のリストアップ。
2.  **動的UIの実装**: チェックを入れた機器に応じて、`file_uploader` やカメラ起動ボタンを増減させる。
3.  **画像リサイズ**: Excelに貼り付ける際、ファイルサイズを抑えるために自動圧縮する。
4.  **Excelテンプレート流し込み**: セル位置を指定したテンプレートExcelへ画像を挿入するロジックの実装。

### 重要な設計判断
- **バリデーション**: 「必須撮影項目」が空のまま「完了」にならないようにガードをかける。
- **保存形式**: 写真のファイル名は「機器名_現場名_日時.jpg」のように、後から目視でも判別可能な形式にする。
- **UI実装**: CSSの `backdrop-filter: blur()`, `linear-gradient` を多用し、現場でも「触りたくなる」プレミアムな操作感を実現する。

### プロンプト案：UIをiOS Liquid Glass風に極める

「添付した『実装指示書』に基づいて実装を進めてください。
特にUIデザインについては、以下の **iOS Liquid Glass (Glassmorphism)** タイルを極限まで追求してください。

1. **デザイン・トーン**: 
   - iOS 17/18 のコントロールセンターのような、透明感と奥行きのあるデザイン。
   - 背景はパステル調のぼかしたグラデーション。
2. **グラス要素**:
   - 各機器のカードや入力エリアは、半透明のガラス（背景ぼかし `backdrop-filter: blur(20px)`）として実装。
   - 縁にはごく細い（1px）の内側光沢（Inner Glow）と、柔らかい影を。
3. **インタラクション**:
   - ボタンなどはクリック・タップ時に液体のような弾力（Elastic）を感じる微妙な拡大縮小アニメーションを追加。
   - チェックを入れた際の「枠の生成」は滑らかなフェードイン・スライドで。
4. **視認性（現場優先）**:
   - ガラスデザインで美しくしつつも、現場での日光下での視認性を損なわないよう、フォントサイズとコントラストは適切に保つ。

まず、このビジュアルコンセプトを反映した **『機器選択画面と撮影スロットのプロトタイプ』** を Python/Streamlit (または指示書内の推奨スタック) でコード出力してください。」
