# ito-photo-manager エージェント向けガイドライン (AGENTS.md)

> 対象: AIエージェント (OpenAI Codex / GitHub Copilot / Cursor など) および開発者  
> 目的: `ito-photo-manager` プロジェクトにおける安全で品質の高い変更を迅速に提供する

## 1. 原則・行動指針

- **影響範囲を最小化**: 変更を加える際は既存の実装と動作への影響範囲を極力狭め、不要なリファクタリングを避ける。
- **Makefileを経由**: インストール・起動・チェック(静的解析・テスト)などのタスクは基本的に `make` コマンドを経由すること。
- **互換性優先**: API 仕様や主要なコンポーネントの挙動を安易に破壊しない。
- **ローカルCI必須**: PRの生成やマージ（あるいはテスト完了通知）を行う前に、必ず `make check` を通過させること。
- **単一ファイル肥大化の抑制**: ビジネスロジックが膨張した場合は、責務ごとにモジュールやコンポーネントを適切に分割する。
- **日本語でのコミュニケーション**: プルリクエスト（PR）やコミットメッセージ、チャットメッセージ等において、言語は基本的に「日本語」で統一すること。

## 2. 開発ワークフローとGitの運用

- **メインブランチへの直接Commit/Push禁止**: いかなる状況であっても `main` への直接 push を禁止する。常に開発用（Topic）ブランチを作成し、PR経由でマージを行うこと。
- **ブランチの整合性確認**: 作業前に、作業対象ブランチが適切（開始点が正しく、名称が意図と一致しているか）であるかを確認する。
- **コミットメッセージ**: `Conventional Commits` 形式に準拠する（例: `feat: add api for user profile`, `fix: error handling logic`）。

## 3. バックエンド(FastAPI) の実装・テスト方針

- **Pydanticによる厳格化**: リクエスト、レスポンスの Body はすべて Pydantic の BaseModel を用いて定義し、`extra="forbid"` で未知フィールドを拒否する。`strict=True` は JSON 経由の date/datetime 変換を阻害するため、リクエスト/レスポンスモデルでは使用しない（`ErrorResponse` 等の内部モデルでは使用可）。
- **レスポンス構造**: `response_model_exclude_none=True` を既定とし、フロントエンドでの無駄な `null` 判定を避ける。また、エラーハンドリングは全APIで統一された例外レスポンスのスキーマを提供する。
- **ビジネスログ**: `print()` によるログ出力は禁止。構造化ログまたは組み込みの `logging` (もしくは指示されたロガー) を使用し、PII (Personally Identifiable Information) の出力を避けること。
- **テスト**: 
  - `make check TARGET=backend`（pytest）を利用すること。
  - レスポンス構造がスキーマから逸脱していないかをテスト（境界値テストを含む）で検証する。

## 4. フロントエンド(Next.js) の実装方針

- **Next.js App Router 規約**: `page.tsx` ファイルはルーティングおよびコンポーネント構成（Composition）に留め、データ取得や複雑な UI 状態管理・ビジネスロジックは別のモジュールやフックへと分割する。
- **命名規則**:
  - React コンポーネントおよび Type/Interface: `PascalCase`
  - ページファイル: `kebab-case`
  - カスタムフックやユーティリティ関数: `camelCase`
  - エクスポート定数: `UPPER_SNAKE_CASE`
- **API通信の一元化**: `fetch` またはラッパーが存在する場合は、必ず共通のインスタンス（例: `lib/api.ts` や共通設定のもの）を使用し、個別のコンポーネント内で直接 `fetch/axios` のハードコードを避けること。

## 5. E2Eテスト方針 (Playwright等)

- スモークテストと結合テストの担保として、`e2e/` フォルダ配下にテストシナリオを構築する。
- テストは `make e2e` の1コマンドで起動・実行できる状態を維持すること。
